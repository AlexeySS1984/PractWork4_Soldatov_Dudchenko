# Рекомендации по тестированию приложения (метод "белого ящика")

## Что такое тестирование "белым ящиком"?

Тестирование "белым ящиком" (white-box testing) - это метод тестирования, при котором тестировщик имеет доступ к исходному коду и внутренней структуре программы.

## Основные задачи при тестировании:

1. **Проверка всех путей выполнения кода**
2. **Тестирование граничных значений**
3. **Проверка обработки исключений**
4. **Валидация входных данных**
5. **Проверка математических вычислений**

## Тестовые сценарии для Страницы 1

### Функция: a = ln(y^(-√|x|)) · (x - y/2) + sin²(arctg(z))

#### 1. Тест на корректные данные:
- **Входные данные:** x = 2, y = 3, z = 1
- **Ожидаемый результат:** Положительное число (проверить вручную)
- **Цель:** Проверка корректности вычислений

#### 2. Тест граничных значений:
- **Входные данные:** x = 0, y = 1, z = 0
- **Ожидаемый результат:** Корректный результат без ошибок
- **Цель:** Проверка работы с нулевыми значениями

#### 3. Тест отрицательных значений:
- **Входные данные:** x = -5, y = 2, z = -3
- **Ожидаемый результат:** Корректный результат
- **Цель:** Проверка работы с отрицательными числами

#### 4. Тест ограничения y ≤ 0:
- **Входные данные:** x = 1, y = 0, z = 1
- **Ожидаемый результат:** Сообщение об ошибке
- **Цель:** Проверка валидации входных данных

#### 5. Тест больших значений:
- **Входные данные:** x = 100, y = 50, z = 20
- **Ожидаемый результат:** Корректный результат или предупреждение
- **Цель:** Проверка на переполнение

#### 6. Тест пустых полей:
- **Входные данные:** Одно или несколько полей пустые
- **Ожидаемый результат:** Сообщение "Заполните все поля"
- **Цель:** Проверка валидации заполненности

#### 7. Тест некорректного формата:
- **Входные данные:** x = "abc", y = 3, z = 1
- **Ожидаемый результат:** Сообщение об ошибке формата
- **Цель:** Проверка обработки исключений

## Тестовые сценарии для Страницы 2

### Функция e (кусочно-заданная)

#### 1. Тест условия "i нечётное, x > 0":
- **Входные данные:** x = 2, i = 3, f(x) = x²
- **Ожидаемый результат:** i√f(x) = 3√4 = 6
- **Цель:** Проверка первого условия

#### 2. Тест условия "i чётное, x < 0":
- **Входные данные:** x = -2, i = 4, f(x) = x²
- **Ожидаемый результат:** (i/2)√|f(x)| = 2√4 = 4
- **Цель:** Проверка второго условия

#### 3. Тест условия "иначе":
- **Входные данные:** x = 0, i = 5, f(x) = x²
- **Ожидаемый результат:** √|if(x)| = 0
- **Цель:** Проверка третьего условия

#### 4. Тест всех функций f(x):
- **sh(x):** Проверить с x = 1
- **x²:** Проверить с x = 3
- **e^x:** Проверить с x = 2
- **Цель:** Проверка всех вариантов функции

#### 5. Тест отрицательного под корнем:
- **Входные данные:** x = 2, i = 3, f(x) вернёт отрицательное
- **Ожидаемый результат:** Сообщение об ошибке
- **Цель:** Проверка математических ограничений

## Тестовые сценарии для Страницы 3

### Функция: y = x⁴ + cos(2 + x³ - b)

#### 1. Тест корректности вычислений:
- **Действие:** Нажать "Вычислить"
- **Ожидаемый результат:** 
  - Таблица со значениями от x=4.6 до x=5.8
  - График отображается корректно
- **Цель:** Проверка основной функциональности

#### 2. Тест количества итераций:
- **Проверка:** Подсчитать количество строк в таблице
- **Ожидаемый результат:** 7 строк ((5.8-4.6)/0.2 + 1 = 7)
- **Цель:** Проверка цикла

#### 3. Тест точности вычислений:
- **Проверка:** Вручную проверить первое и последнее значение
- **x = 4.6:** y = 4.6⁴ + cos(2 + 4.6³ - 1.3)
- **x = 5.8:** y = 5.8⁴ + cos(2 + 5.8³ - 1.3)
- **Цель:** Проверка точности формулы

#### 4. Тест графика:
- **Проверка:** 
  - График строится без ошибок
  - Оси подписаны корректно
  - Точки соответствуют значениям из таблицы
- **Цель:** Проверка визуализации

#### 5. Тест кнопки "Очистить":
- **Действие:** Вычислить, затем нажать "Очистить"
- **Ожидаемый результат:** Таблица и график очищены
- **Цель:** Проверка функции очистки

## Общие тесты для всех страниц

### 1. Тест навигации:
- Переход между страницами работает корректно
- Данные не теряются при переходе назад

### 2. Тест кнопки "Очистить":
- Все поля очищаются
- Фокус переходит на первое поле ввода

### 3. Тест всплывающих подсказок:
- Все элементы имеют ToolTip
- Подсказки понятны и информативны

### 4. Тест выхода из приложения:
- При нажатии "Выход" появляется диалог подтверждения
- При закрытии окна также появляется диалог

### 5. Тест блокировки полей результата:
- Поля с результатами нельзя редактировать вручную

## Чек-лист тестирования

### Функциональное тестирование:
- [ ] Все математические функции вычисляются корректно
- [ ] Валидация входных данных работает
- [ ] Обработка ошибок реализована
- [ ] Все кнопки функционируют

### UI тестирование:
- [ ] Интерфейс отзывчивый и понятный
- [ ] Всплывающие подсказки присутствуют
- [ ] Формулы отображаются (изображения)
- [ ] График строится корректно

### Тестирование граничных значений:
- [ ] Нулевые значения
- [ ] Отрицательные значения

- [ ] Очень маленькие значения

- [ ] Пустые поля
- [ ] Некорректный формат данных
- [ ] Недопустимые значения (y ≤ 0)
- [ ] Значения вне области определения

### Тестирование производительности:
- [ ] Приложение запускается быстро
- [ ] Вычисления выполняются без задержек
- [ ] График строится мгновенно

## Оформление результатов тестирования

### Формат отчёта о тесте:

```
Тест №: 1
Название: Проверка корректности вычисления функции a
Дата: 14.02.2026
Тестировщик: Солдатов

Шаги:
1. Открыть страницу 1
2. Ввести x = 2, y = 3, z = 1
3. Нажать "Вычислить"

Ожидаемый результат: 
Положительное число, математически корректное

Фактический результат:
a = 4.9738570564

Статус: ✅ PASSED

Комментарии:
Вычисление выполнено корректно, результат соответствует ожидаемому.
```

## Типичные ошибки, которые нужно найти

1. ❌ Отсутствие валидации входных данных
2. ❌ Неправильная обработка исключений
3. ❌ Ошибки в математических формулах
4. ❌ Неверные граничные условия
5. ❌ Отсутствие проверки деления на ноль
6. ❌ Некорректная работа с отрицательными числами под корнем
7. ❌ Отсутствие блокировки полей результата
