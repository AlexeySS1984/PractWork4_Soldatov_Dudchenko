# Инструкция по работе с Git для проекта

## 1. Создание репозитория в Visual Studio

1. Откройте проект в Visual Studio
2. В меню выберите: **Git → Создать репозиторий Git...**
3. Войдите в свою учетную запись GitHub
4. Уберите галочку "Частный репозиторий"
5. Нажмите "Создать и отправить"

## 2. Фиксация изменений

После внесения изменений в проект:

1. Перейдите в меню **Git → Фиксация или скрытие**
2. Напишите комментарий (коммит), описывающий изменения
3. Нажмите **"Зафиксировать все"**
4. Нажмите кнопку **"↑ Push"** или выберите **Git → Отправить**

## 3. Добавление соразработчика

1. Перейдите на сайт GitHub.com
2. Откройте ваш репозиторий
3. Перейдите в **Settings → Collaborators**
4. Добавьте аккаунт **TGAksenova** как соразработчика

## 4. Клонирование репозитория

Если вы работаете в паре и хотите клонировать проект:

1. В Visual Studio выберите **"Клонирование репозитория"**
2. Введите URL: `https://github.com/ваш_логин/PractWork4_Soldatov_Dudchenko`
3. Выберите пустую папку для клонирования
4. Нажмите "Клонировать"

## 5. Совместная работа

При работе в паре:

1. Разделите работу по страницам (каждый работает над разными файлами)
2. Регулярно фиксируйте изменения (коммиты)
3. Отправляйте изменения в репозиторий (Push)
4. Получайте изменения коллеги (Pull)

### Команды для синхронизации:

- **Git → Получить (Pull)** - получить и объединить изменения
- **Git → Извлечь (Fetch)** - только проверить наличие изменений
- **Git → Отправить (Push)** - отправить свои изменения

## 6. Просмотр истории

Для просмотра всех коммитов:

**Git → Просмотреть журнал ветвей**

Здесь вы увидите:
- Все коммиты
- Автора каждого коммита
- Дату и время изменений
- Комментарий к изменениям

## Полезные команды Git в командной строке

```bash
# Инициализация репозитория
git init

# Добавление всех файлов
git add .

# Создание коммита
git commit -m "Описание изменений"

# Связывание с удалённым репозиторием
git remote add origin https://github.com/username/repo.git

# Отправка изменений
git push -u origin master

# Получение изменений
git pull origin master

# Просмотр статуса
git status

# Просмотр истории
git log
```

## Важно!

- **Всегда пишите осмысленные комментарии к коммитам**
- **Регулярно сохраняйте изменения**
- **Перед началом работы делайте Pull, чтобы получить последнюю версию**
- **Не забывайте отправлять изменения (Push) после работы**

## Типичные ошибки и их решение

### Конфликт слияния
Если вы и ваш коллега изменили один и тот же файл:
1. Visual Studio покажет конфликт
2. Откройте файл и выберите нужные изменения
3. Зафиксируйте разрешение конфликта

### Забыли сделать Pull
Если Push не удаётся:
1. Сделайте Pull
2. Разрешите конфликты (если есть)
3. Снова сделайте Push

### Нет доступа к репозиторию
1. Проверьте, что вы добавлены как соразработчик
2. Проверьте логин и пароль GitHub
3. Убедитесь, что репозиторий публичный
